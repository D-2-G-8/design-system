# @d-2-g-8/design-system

Shared React component library. Components and design tokens here are
generated and kept in sync from Figma by
[ai-tools-app](https://github.com/D-2-G-8/ai-tools-app)'s Design System
sync (`/design-system/settings` in that app) -- this repo is the
destination for that generated code, not where you hand-author new
components day to day. See `ai-tools-app`'s README ("Design system code
sync") for how a Figma sync turns into a pull request here.

## What's here

- `src/components/<slug>/` -- one folder per component: `Component.tsx`,
  `Component.module.css` (CSS Modules), `Component.stories.tsx`. Every
  generated component follows this exact shape; `src/components/button/`
  is a hand-written example proving the pipeline before any real
  Figma-generated component exists.
- `src/tokens/tokens.css` -- design tokens as CSS custom properties
  (`--token-name`). Components reference tokens via `var(--token-name)`,
  never a hardcoded value, so a token resync updates every component at
  once. Regenerated by a deterministic serializer in `ai-tools-app`, not
  hand-edited.
- `.storybook/` -- local component preview, and the same `.stories.tsx`
  files this builds from are what `ai-tools-app`'s component detail page
  iframes for its own live preview (see that repo's README).

## Consuming this package

This publishes to GitHub Packages, not the public npm registry -- installs
need a `read:packages`-scoped GitHub token.

1. Add to your project's `.npmrc`:
   ```
   @d-2-g-8:registry=https://npm.pkg.github.com
   //npm.pkg.github.com/:_authToken=${GITHUB_PACKAGES_READ_TOKEN}
   ```
2. Set `GITHUB_PACKAGES_READ_TOKEN` (a GitHub personal access token with
   `read:packages`) in your environment.
3. `npm install @d-2-g-8/design-system` (or the pnpm/yarn equivalent).
4. Import the design tokens once, globally, alongside the components:
   ```ts
   import "@d-2-g-8/design-system/tokens.css";
   import "@d-2-g-8/design-system/style.css"; // component styles (dist/design-system.css under the hood)
   import { Button } from "@d-2-g-8/design-system";
   ```

## Local development

```sh
pnpm install
pnpm run storybook       # preview components at localhost:6006
pnpm run build            # library build (Vite, not tsup -- see vite.config.ts
                           # for why: CSS Modules need a real CSS-Modules-aware
                           # bundler, which esbuild-based tsup doesn't provide
                           # out of the box)
pnpm run typecheck
pnpm run lint
```

## Publishing

Push to `master` (normally via a merged pull request from `ai-tools-app`'s
sync) triggers `.github/workflows/publish.yml`: typecheck, lint, build,
build Storybook, bump the patch version, and `npm publish` to GitHub
Packages. Nothing to run by hand.

## Styling approach

v1 only supports CSS Modules (`workspace.designComponentStack ===
"react-css-modules"` in `ai-tools-app`) -- plain `.module.css` per
component, tokens as CSS custom properties. SCSS is a selectable-but-
not-yet-implemented option on the `ai-tools-app` side.
